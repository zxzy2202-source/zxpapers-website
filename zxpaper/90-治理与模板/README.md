# 90 治理与模板

## 正式文档最低元数据

```yaml
title: 文档标题
product_lines: []
dimension: Product | Market | Persona | Sales | PageOps | Evidence | Governance
audience: []
status: Draft | Review | Active | Needs Validation | Needs Update | Archive
owner: 待指定
sources: []
write_back_to: []
last_reviewed: YYYY-MM-DD
next_review: YYYY-MM-DD
```

## 新建文档门槛

同时满足以下条件才新建正式文档：

1. 对应长期存在的对象或任务。
2. 与现有文档至少有两个独立差异，如决策链、使用环境、规格、合规或搜索意图。
3. 有明确使用者、Owner 和回写位置。
4. 至少有一条可定位来源；否则保留为待验证素材。
5. 能说明何时复查，以及失效后如何处理。

## 治理规则

- 禁止用“最终版、最新版、新文档、作者名”作为正式标题。
- 检查报告只记录问题、Owner 和整改状态，不复制正式结论。
- 角色工作台只做任务导航，不复制产品、客户或证据正文。
- 每月复查销售/客户内容，每季度复查网站与 SEO/GEO，每半年复查市场与行业。
- 飞书写入前先生成迁移计划和准确范围；移动、创建或归档后必须 fresh read 验证。
