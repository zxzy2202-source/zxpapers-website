# ZXPaper 知识库总索引

## 1. 按角色进入

| 角色 | 第一入口 | 典型任务 |
|---|---|---|
| 老板 / 负责人 | `20-市场与客户`、`50-证据与合规` | 产品线优先级、市场选择、能力边界 |
| 销售 | `30-销售与交付` | 询盘判断、报价追问、样品确认、异议处理 |
| 产品 / QC | `10-产品知识包`、`50-证据与合规` | 规格选型、适配、测试、声明审核 |
| 网站运营 | `40-网站与增长` | URL 归属、页面 Brief、发布与复盘 |
| SEO / GEO | `40-网站与增长`、`50-证据与合规` | 关键词、FAQ、可引用答案、来源核验 |
| 新人 | 本页 -> `10-产品知识包` -> `30-销售与交付` | 建立产品与业务流程全景 |

## 2. 按任务进入

| 我现在要做什么 | 去哪里 |
|---|---|
| 判断客户是否值得跟进 | `30-销售与交付/线索资格` |
| 确认规格、材料、尺寸和设备适配 | `10-产品知识包` 对应产品线 |
| 找目标客户、行业或区域机会 | `20-市场与客户` |
| 准备报价、样品或试机 | `30-销售与交付` + `10-产品知识包` |
| 新建或改版产品页 | `40-网站与增长` + 对应产品知识包 |
| 判断一句卖点能不能对外使用 | `50-证据与合规` |
| 新增研究或文档 | `90-治理与模板` |

## 4. 内容状态

| 状态 | 含义 | 可用于 |
|---|---|---|
| Draft | 初稿，尚未评审 | 内部讨论 |
| Review | 已成形，待业务或技术审核 | 页面/销售准备 |
| Active | 当前有效 | 销售、运营、培训 |
| Needs Validation | 关键事实或能力尚缺证据 | 研究参考，不可直接承诺 |
| Needs Update | 已知可能过期 | 暂停复用，安排复查 |
| Archive | 历史底稿或旧版 | 溯源，不作为日常入口 |

## 5. 内容流转

```text
原始研究 / 客户原话 / 测试记录
              ↓
市场与客户 / 产品知识 / 证据登记
              ↓
销售工具与网站 Brief
              ↓
页面、报价、样品、客户沟通
              ↓
询盘与项目反馈回写
```
