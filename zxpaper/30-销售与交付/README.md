# 30 销售与交付

这里把研究转成销售可以直接执行的检查表、问法和交付节点。

## 标准结构

| 模块 | 核心资产 |
|---|---|
| `31-线索资格` | 资格评分、红旗、排除条件、下一步动作 |
| `32-需求诊断` | 按产品线的首轮追问和技术升级条件 |
| `33-报价与成本输入` | 必填规格、数量、包装、贸易条款、交期 |
| `34-样品与试机` | 样品计划、实机测试、签样、失败处理 |
| `35-方案与异议处理` | 痛点到方案、价格/质量/交期/合规异议 |
| `36-项目交付` | 打样、生产、QC、包装、出运、复盘节点 |
| `37-复购与交叉销售` | 补货信号、产品组合、客户反馈回写 |

## 使用规则

- 销售工具引用产品事实时链接到 `10-产品知识包`，不在话术里维护另一套规格。
- 所有性能、合规和能力陈述引用 `50-证据与合规` 的证据 ID。
- 客户反馈先作为原始事实记录，验证后再回写画像、产品或页面资产。
- P0 首批应覆盖四条产品线各自的“资格审查 + 报价前问询 + 打样/试机确认”三件套。
