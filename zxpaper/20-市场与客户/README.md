# 20 市场与客户

这里维护“卖给谁、为什么买、何时买”的判断，不重复产品规格正文。

## 标准结构

| 模块 | 内容 | 主要输出 |
|---|---|---|
| `21-ICP与客户优先级` | P0/P1/P2 客户、排除条件、评分字段 | 线索筛选规则 |
| `22-买家角色与决策链` | 使用者、技术影响者、采购、老板 | 角色话术与推进路径 |
| `23-行业与应用场景` | 零售、物流、医疗、食品、工业、政务等 | 行业机会卡 |
| `24-区域与国家` | 需求逻辑、法规、渠道和竞争环境 | 市场优先级 |
| `25-JTBD与触发信号` | 换供应商、扩店、合规变化、设备更换等 | 开发信号清单 |
| `26-VoC与反对意见` | 客户原话、痛点、风险和购买障碍 | 页面/销售输入 |
| `27-目标账户` | 具名账户、公开信号、状态和下一步 | 账户队列 |

## 迁移规则

1. 旧库的 `B-客户画像`、JTBD、ICP 和行业专题在此唯一归档。
2. 同一客户画像跨产品线可复用时，建一份跨品类画像，并用字段标明适用产品线。
3. 品牌或企业名称只能表示目标账户或公开场景原型，不能暗示客户关系。
4. 日期敏感的市场、法规和搜索量数据必须记录来源日期与复查日期。
