How should the three template families be implemented?

All three approaches preserve the current page designs. The difference is how much code becomes reusable and how safely future pages can be created.

2. Composable typed templates (recommended)

Three clear templates share small primitives and builders. Each new page supplies typed content data and enables only relevant modules. Strong consistency without forcing all pages into one layout.

3. Schema-driven renderer

Every section is described as JSON and rendered dynamically. Maximum flexibility, but harder TypeScript contracts, debugging, visual governance and migration effort for the current site.

Recommended outcome

  • Existing pages remain visually recognizable
  • New pages are configuration-first
  • SEO/GEO and evidence fields become required by type
  • Optional modules prevent irrelevant content

Deliberately avoided

  • One giant component with dozens of conditionals
  • Freeform section order without governance
  • Immediate migration of every existing route
  • Copying 500+ line JSX files for each product
