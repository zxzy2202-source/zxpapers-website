## 一、🟢 显著优势

### 1.1 Metadata 基础设施 — 95分

根布局 `layout.tsx` 使用 `generateMetadata()` 动态生成全局元数据，支持后台 SEO 面板实时覆盖：

- **title template**: `%s | ZhixinPaper` — 子页标题自动拼接品牌名 ([layout.tsx:77](src/app/layout.tsx#L77))
- **metadataBase**: `new URL(SITE.domain)` — 正确设置了基 URL ([layout.tsx:95](src/app/layout.tsx#L95))
- **Open Graph**: type=website, siteName, locale=en_US, 1200x630 图片 ([layout.tsx:112](src/app/layout.tsx#L112))
- **Twitter Card**: summary_large_image ([layout.tsx:129](src/app/layout.tsx#L129))
- **robots**: index/follow + googleBot 详细配置，max-image-preview:large, max-snippet:-1 ([layout.tsx:135](src/app/layout.tsx#L135))
- **站点验证**: Google + Bing + Baidu 三站验证支持 ([layout.tsx:85](src/app/layout.tsx#L85))
- **自动回退**: 后台 SEO 未配置时，自动使用 `SITE.name` 和 `SITE.tagline` 兜底 ([layout.tsx:75](src/app/layout.tsx#L75))

每个子页面均导出独立的 `metadata` 或 `generateMetadata()`，覆盖了 title、description、keywords、canonical、OG。

### 1.2 结构化数据 — 90分

**全站级（layout.tsx 注入所有页面）：**
- `Organization` + `Manufacturer` — 含 legalName、alternateName、foundingDate、address、contactPoint、sameAs、hasCredential、hasOfferCatalog ([layout.tsx:149](src/app/layout.tsx#L149))
- `WebSite` — 含 publisher 引用 ([layout.tsx:249](src/app/layout.tsx#L249))
- **亮点**: OfferCatalog 中每个产品目录项都包含完整 URL，但不含 Product 实体（避免了无价格/评论时的 Search Console 警告）

**页面级（覆盖 30+ 页面）：**

| Schema 类型 | 覆盖页面数 |
|-------------|-----------|
| BreadcrumbList | 30+ 页面 |
| FAQPage | 产品分类页、详情页、对比页 |

### 1.3 Sitemap — 95分

[sitemap.ts](src/app/sitemap.ts) 实现了覆盖 100+ URL 的动态 sitemap：

- **ISR**: `revalidate = 3600`，每小时重新生成
- **URL 去重**: `uniqueByUrl()` 函数
- **优先级分层**: 首页 1.0 → 核心产品 0.9 → 二级产品 0.85 → 市场 0.85 → OEM 0.75 → 博客 0.6 → 国家页 0.5
- **changeFrequency**: 合理分配 weekly/monthly
- **动态博客**: 从 KV 存储读取已发布文章，自动包含 `lastModified`

### 1.4 重定向策略 — 95分

[next.config.ts](next.config.ts) 包含 80+ 条重定向规则：

- WordPress 产品 slug → Next.js 规范路径（43 条精确匹配）
- 19 种语言的 `/lang/product/…` 和 `/lang/…/` 模式全部 301
- 产品别名 → 规范内容页（如 `/products/custom-printed-rolls` → `/products/thermal-paper-rolls/custom-printed`）
- 域名统一：`zxpapers.com` → `www.zxpapers.com`，`thermalrollpro.com` → `www.zxpapers.com`
- 欧洲市场 → `zhixinpaper.com/eu`（区域所有权分离）
- 旧路径：`/about-us`、`/contact-us`、`/inquiry`、`/get-a-quote`、`/category/*`、`/tag/*`、`/feed` 等全部映射

### 1.5 GEO / AI 可见性 — 95分

**robots.txt** 是业内最佳实践级别的 GEO 配置：

- 为 9 个 AI crawler 单独声明 Allow（GPTBot、ChatGPT-User、OAI-SearchBot、ClaudeBot、Claude-Web、anthropic-ai、PerplexityBot、Google-Extended、Applebot-Extended、Bingbot、CCBot、Amazonbot）
- 每个 AI crawler 都 Disallow `/admin/` 和 `/api/`
- admin 路径同时对人类爬虫和 AI 爬虫封闭

**llms.txt** ([public/llms.txt](public/llms.txt)) 提供了 113 行的结构化工厂数据、产品目录、市场覆盖、联系信息，对 LLM 训练和 RAG 检索非常友好。

**问题点：**

- 给 `PageHero` 增加可选的 `bgImageAlt` prop，从页面配置传入描述性文本

## 三、🔴 需要修复

### 3.1 hreflang / 国际化 — 40分（唯一重大缺口）

**现状：**
- 全站无任何 `hreflang` 标签或 `alternates.languages` 配置
- `zhixinpaper.com` 和 `zxpapers.com` 是独立部署，各自缺少对对方语言版本的声明

**为什么重要：**
- 该站曾有 19 种语言的 WordPress 页面，Google 索引中可能仍有残留
- 缺少 hreflang 意味着 Google 无法确定：
  - `zhixinpaper.com/eu` vs `zxpapers.com/markets/africa` 是不同区域的页面，不是重复内容
  - 哪个域名是英语版本的权威来源
- 可能导致 zhixinpaper.com 和 zxpapers.com 在搜索结果中互相竞争

**建议方案：**
1. 在 `layout.tsx` 的 `generateMetadata()` 中添加 `alternates.languages: { 'en': SITE.domain }`
2. 在每个产品/市场页面的 metadata 中加入 `alternates.languages`
3. 如果 zhixinpaper.com 上的欧洲页面曾使用其他语言（如 `zhixinpaper.com/it/`），考虑添加对应的 hreflang 声明

## 简短结论

这个网站的 SEO 工程基础属于 **B2B 外贸站中的一流水平**。结构化数据、sitemap、robots.txt（含 AI crawler 策略）、llms.txt、重定向策略都做到了"无需再改"的程度。唯一的实质性缺口是 **hreflang 完全缺失**——修复这一点（改 2 行代码）即可把 SEO 得分拉到 92+。图片 LCP 优化是下一步性能改进的主要方向。
