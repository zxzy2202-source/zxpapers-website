## 一、🟢 显著优势

### 1.1 设计系统基础扎实

[tailwind.config.ts](tailwind.config.ts) 和 [globals.css](src/app/globals.css) 定义了完整的设计令牌体系：

- **品牌色板**：从 `brand-ink` (#07172f) 到 `brand-navy` (#0F2B5B) 的深蓝渐变阶梯，搭配 `brand-amber` (#F59E0B) 强调色
- **shadcn/ui 变量**：标准的 CSS 变量体系（--primary, --secondary, --muted 等）
- **字体系统**：Sora（标题）+ Inter（正文），通过 Next.js font loader 加载，`display: swap`
- **组件工具类**：`.btn-primary` / `.btn-secondary` / `.btn-navy` / `.card-hover` / `.section-title` / `.badge-amber`

### 1.2 交互状态覆盖完整 — 88分

全站所有可交互元素都覆盖了以下状态：

| 状态 | 覆盖情况 |
|------|----------|
| hover | 100% — 所有链接、按钮均有 `hover:bg-*` / `hover:text-*` / `hover:border-*` |
| focus-visible | 90%+ — 全局 `.focus-visible` 配置 + 元素级 `focus-visible:ring-2` |
| disabled | Button 组件有 `disabled:pointer-events-none disabled:opacity-50` |
| active | Footer 按钮有 `active:translate-y-px` |
| aria-expanded | Header dropdown + 移动端菜单均绑定 |
| aria-label | 汉堡按钮、WhatsApp FAB、图标按钮均标注 |

WhatsApp FAB 是交互最佳实践的标杆实现：
- Escape 键关闭面板并返回焦点到触发按钮
- 打开时自动聚焦面板内第一个链接
- 脉冲环动画 + 悬停绿色切换
- 面板内快捷消息可一键跳转 WhatsApp

### 1.3 过渡与动画系统性 — 90分

- 所有可交互元素统一使用 `transition-colors duration-200`
- 卡片悬停有 `hover:-translate-y-0.5` / `hover:shadow-lg`
- 图片有 `group-hover:scale-105` + `duration-500`
- 箭头图标有 `group-hover:translate-x-0.5` / `group-hover:gap-2.5`
- 手风琴展开使用 Radix 原生 `animate-accordion-down/up`
- 反馈轮播有 `animate-fade-up` + `feedbackCarousel` 自动滚动
- **所有动画尊重 `prefers-reduced-motion`**：`motion-reduce:transition-none`

### 1.4 InquiryForm 表单 UX 优秀 — 90分

- 完整的状态流转：idle → submitting → success → error
- 后端校验 + 前端即时校验（红色高亮 + 错误文案）
- GEO 自动填充国家栏（`/api/geo` + sessionStorage 缓存）
- `details/summary` 可折叠的可选字段（公司名/电话）
- 提交按钮 loading spinner + 禁用态
- 成功态有 CheckCircle 图标 + "Send another inquiry" 返回链接
- 错误态显示 fallback 邮箱联系方式
- 转化漏斗追踪：`inquiry_form_started` → `validation_failed` → `submit_success/failed`

## 二、🟡 可优化项

### 2.1 圆角策略不一致 — 影响 65 分的组件一致性

项目中圆角值缺乏统一标准，同一页面内混用多种：

| 圆角值 | 使用位置示例 |
|--------|-------------|
| `rounded-md` | 全局 Button 组件、Header 导航链接、Footer 按钮 |
| `rounded-lg` | WhatsApp FAB 面板 |
| `rounded-xl` | ProductLandingPageTemplate 中多处卡片、SizeDetailPage 事实标签 |
| `rounded-2xl` | MarketCountryPageTemplate 卡片、ProductsCatalogPage |
| `rounded-[22px]` | ProductLandingPageTemplate 图片 |
| `rounded-[24px]` | ProductCategoryShowcaseTemplate 特性卡片 |
| `rounded-[26px]` | ProductCategoryShowcaseTemplate 询盘卡片、FAQ 卡片 |
| `rounded-[28px]` | ProductCategoryShowcaseTemplate Hero 面板、CTA 底部 |

**建议**：统一为 3-4 级圆角方案：
- 内联按钮/标签：`rounded-md`（6px）
- 卡片：`rounded-xl`（12px）
- Hero 面板：`rounded-2xl`（16px）
- 统一删除 `rounded-[22px]` / `[24px]` / `[26px]` / `[28px]` 等魔法值

### 2.2 按钮组件碎片化

项目中存在 3 套按钮体系，混用导致视觉不一致：

1. **`<Button>` 组件**（shadcn/ui）— `rounded-md`，用得较少
2. **CSS 工具类**（`.btn-primary` / `.btn-navy`）— `rounded-md`，定义在 globals.css
3. **内联 Tailwind**（如 Header CTA、PageHero CTAs）— 手工复刻，未引用工具类

同一 CTA 在不同上下文表现为不同形状：
- Footer "Request a Quote"：`bg-amber-500 rounded-md`
- Market country "Get CIF Quote"：`bg-amber-500 rounded-xl`  
- Header "Request a Quote"：`bg-brand-navy rounded-md`

**建议**：将所有 CTA 统一回 `<Button variant="amber" size="cta">` / `<Button variant="navy">`，删除重复的 CSS 工具类。

### 2.3 卡片悬停色不一致

同一类型卡片在不同模板中的 hover 态各不同：

| 所在模板 | 当前 hover | 
|----------|-----------|
| MarketCountryPageTemplate | `hover:border-blue-200` |
| ProductCategoryShowcaseTemplate | `hover:border-amber-300` |
| LatestArticles | `hover:border-brand-navy` |
| SizeDetailPage | `hover:border-brand-navy` |
| ProductsCatalogPage | `hover:bg-amber-50` |

**建议**：统一为 `hover:border-brand-navy/30 hover:shadow-md`（品牌深蓝边框 + 微投影）作为全局卡片 hover 标准。

### 2.4 移动端断点过高

桌面导航在 `xl`（1280px）以下隐藏，转换为汉堡菜单：
```tsx
// src/components/layout/Header.tsx:207
<div className="relative hidden xl:flex items-center ...">
```

| 断点 | 设备范围 | 当前体验 |
|------|----------|----------|
| < 640px | 手机 | 汉堡菜单 ✅ |
| 640-1023px | 平板竖屏 + 小笔记本 | 汉堡菜单 ⚠️ 空间足够放简化导航 |
| 1024-1279px | 平板横屏 + 普通笔记本 | 汉堡菜单 ❌ 空间完全够放桌面导航 |
| ≥ 1280px | 大屏笔记本 + 桌面 | 完整桌面导航 ✅ |

**建议**：将断点从 `xl`（1280px）下探到至少 `lg`（1024px），让 iPad Pro 横屏和 13寸笔记本用户能直接看到导航。

### 2.5 Top bar 移动端占用首屏

移动端 top bar 显示电话+邮箱（[Header.tsx:167-174](src/components/layout/Header.tsx#L167)），而认证信息（ISO 9001 / Global Shipping）在 `sm` 以下隐藏。实际结果是：**手机首屏约 25px 高度被电话和邮箱占据**，但 B2B 访客最需要的信任信号（认证、产能）反而不显示。

**建议**：移动端 top bar 只保留一个 WhatsApp/询盘入口，释放首屏空间给 Hero。认证信息移到 Hero 的 trustBadges 或面包屑下方。

## 三、🔴 需修复

### 3.1 Footer Trust Bar 视觉割裂

[Footer.tsx:95-108](src/components/layout/Footer.tsx#L95) 的信任图标区域使用了**无圆角的方形图标容器**：
```tsx
<span className="flex h-9 w-9 flex-none items-center justify-center border border-white/10 bg-white/5 text-amber-300">
```

而全站其他地方的图标/徽章都有圆角（`rounded-md` / `rounded-xl`）。这个 90° 直角在深色背景上形成强烈对比，与整体圆角设计语言脱节。

**建议**：加上 `rounded-lg` 或直接复用全局徽章样式。

### 3.2 Footer 企业名称前面的图标容器缺少圆角

[Footer.tsx:98](src/components/layout/Footer.tsx#L98) 的图标容器样式缺少 `rounded` 类，呈现方形，与全局圆角风格不统一。只需加 `rounded-md` 即可修复。

### 3.3 首页 `py-20` 扁平节奏（已有报告提及）

首页 10 个 section 大部分使用 `py-20`（80px 上下），缺少节奏变化：

| Section | 当前 padding | 建议 |
|---------|-------------|------|
| Hero | `min-h-[560px]` | ✅ |
| 产品线卡片 | `py-20` | → `py-16` |
| 信任条 | `py-12` | ✅ |
| 工厂数据 | `py-20` | → `py-16` |
| 深色 CTA 段 | `py-20` | → `py-16` |
| OEM 能力 | `py-20` | → `py-14` |

## 五、优先级建议

### 🔴 现在处理（低改动、高视觉收益）

1. **统一圆角** — 删除 `rounded-[22px]/[24px]/[26px]/[28px]` 魔法值，统一为 `rounded-md` / `rounded-xl` / `rounded-2xl` 三级
2. **Footer 图标加圆角** — Trust bar 图标容器加 `rounded-lg`
3. **导航断点下探** — `xl` → `lg`（1280px → 1024px），让 iPad Pro 和 13寸笔记本用户看到桌面导航

### 🟡 第二优先（提升一致性）

4. **统一 Button 体系** — 用 `<Button variant>` 替代内联 Tailwind 按钮
5. **统一卡片 hover** — 固定为 `hover:border-brand-navy/30`
6. **移动端 top bar 精简** — 只保留询盘入口
7. **首页 spacing 节奏** — 给 section 建立 `py-12/14/16/20` 分层

### 🟢 锦上添花

8. 添加页面级加载骨架屏（首页 + 产品目录页）
9. 评估暗黑模式需求（B2B 外贸站通常不需要，可删除未使用的 darkMode 配置以减少困惑）

## 简短结论

这个网站的 UI 工程质量属于 B2B 外贸站中**中上水平**。交互状态覆盖、动画系统、表单 UX、可访问性都做得很好，WhatsApp FAB 和 InquiryForm 尤其出色。

当前主要问题是**设计碎片化**——圆角、按钮、卡片 hover 各有 3-7 种变体，是典型的"功能迭代快于设计规范"的产物。统一这些（改 CSS 类名即可，约 20 处）能以最小成本大幅提升专业感。
