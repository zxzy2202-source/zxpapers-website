### 🔴 BUG #1 — GA / GTM 完全没生效

老板在后台填了 `G-XXXXXXX` 或 `GTM-XXXXXX`，**前端不会装载 Google Analytics 脚本**，数据完全收不到。

**修复方法**（10 行代码）：在 `src/app/layout.tsx` `<body>` 末尾加：

```tsx
const adminSeo = await readSeo().catch(() => ({} as Awaited<ReturnType<typeof readSeo>>));
// ... layout 渲染时：
{adminSeo.googleAnalyticsId && (
  <>
    <Script src={`https://www.googletagmanager.com/gtag/js?id=${adminSeo.googleAnalyticsId}`} strategy="afterInteractive" />
    <Script id="ga" strategy="afterInteractive">
      {`window.dataLayer=window.dataLayer||[];function gtag(){dataLayer.push(arguments);}gtag('js',new Date());gtag('config','${adminSeo.googleAnalyticsId}');`}
    </Script>
  </>
)}
{adminSeo.googleTagManagerId && (
  <Script id="gtm" strategy="afterInteractive">
    {`(function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'gtm.start':new Date().getTime(),event:'gtm.js'});var f=d.getElementsByTagName(s)[0],j=d.createElement(s),dl=l!='dataLayer'?'&l='+l:'';j.async=true;j.src='https://www.googletagmanager.com/gtm.js?id='+i+dl;f.parentNode.insertBefore(j,f);})(window,document,'script','dataLayer','${adminSeo.googleTagManagerId}');`}
  </Script>
)}
```

## 七、根因分析

### 为什么会出现这么多不一致？

1. **IMAGE_SLOTS 是事后补写的**：代码先有了 `getSlotImage("xxx:yyy")` 调用，后才加注册表，没人对齐。
2. **没有编译期检查**：TypeScript 没把 slot 设成 union 类型，传任何字符串都不报错。
3. **没有自动同步机制**：注册表是手写 markdown 表格那种数据结构，跟代码完全脱钩。

### 如何彻底根治（架构层面）

**方案 1（轻量）**：把 IMAGE_SLOTS 的 `slot` 字段抽成 TypeScript union 类型：

```typescript
// imageSlots.ts
export const SLOT_KEYS = ["home:hero", "home:hero-slide-2", /* ... */] as const;
export type SlotKey = typeof SLOT_KEYS[number];

// imageSlotUtils.ts
export function getSlotImage(slot: SlotKey, fallback: string) { ... }
```

→ 调用 `getSlotImage("typo:wrong")` 会**编译报错**，再也不可能错位。

**方案 2（彻底）**：写一个 codemod 扫描全站，把所有 `getSlotImage("xxx")` 调用自动收集，比对注册表，**自动生成同步的注册表**，跑 `pnpm sync:slots` 一键修复。
